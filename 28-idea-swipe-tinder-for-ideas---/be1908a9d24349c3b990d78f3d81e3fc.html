Real-time Diff Sync for Distributed Markdown Editors

terminal Infrastucture Added 2 days ago timer 4m read

The core challenge lies in minimizing conflict resolution latency while maintaining absolute state integrity across high-latency peer-to-peer connections. Current solutions like Yjs or Automerge provide robust CRDT foundations, but often feel too heavy for lightweight markdown implementations.

This idea proposes a hybrid approach: using a simplified Diff-Match-Patch algorithm synchronized via a central relay for "soft-locks" on specific block-level elements (paragraphs, list items) to prevent simultaneous edits, effectively creating a "seamless pessimistic" locking system that feels optimistic to the user.

Technical Constraint:

Must support offline editing with re-syncing capabilities that prioritize the latest timestamped changes at the block level rather than line level.
